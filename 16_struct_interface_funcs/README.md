# Практическая работа по структурам, интерфейсам и функциям в Go

## Бизнес-задача
>Вам необходимо реализовать Go модуль, который будет работать с платежами.

Пользователь вашего пакета должен иметь возможность получить из исходных данных о платежах следующую информацию:
- сумму всех платежей
- среднюю сумму платежа
- выручку за период
- общую выручку

## Входные данные
Файл в формате json с данными о платежах. 
Содержимое файла представляет собой массив объектов, каждый из которых содержит следующие поля:
- `id` - идентификатор платежа (всегда строка). Обязательное поле.
- `type` - тип платежа (всегда строка). Обязательное поле. Возможные значения: `income`, `outcome`.
- `timestamp` - временная метка платежа в unix формате (всегда целое число). Необязательное поле.
- `datetime` - временная метка платежа в формате `YYYY-MM-DDTHH:MM:SS` (всегда строка). Необязательное поле.
- `sum` - сумма платежа (всегда целое число). Необязательное поле.
- `amount` - сумма платежа (всегда целое число). Необязательное поле.
- `bank` - банк, через который прошел платеж (всегда строка). Обязательное поле. Возможные значения: `Sber`, `T-Bank`.

Для банка `Sber` используется поле `sum` и `timestamp`
Для банка `T-Bank` - `amount` и `datetime`

Пример файла:
```json
[
    {
        "id": "1",
        "timestamp": 1727921347,
        "sum": 384,
        "type": "income",
        "bank": "Sber"
    },
    {
        "id": "aaaa-bbbb-cccc-dddd",
        "datetime": "2021-01-01T00:00:00",
        "amount": 1190,
        "type": "outcome",
        "bank": "T-Bank"
    }
]
```

## Техническое задание
1. Создайте структуру `TBankPayment` с полями:
    - `Id` - идентификатор платежа
    - `CreatedAt` - временная метка платежа
    - `Amount` - сумма платежа

2. Создайте структуру `SberPayment` с полями:
   - `Id` - идентификатор платежа
   - `CreatedAt` - временная метка платежа
   - `Sum` - сумма платежа

3. Создайте интерфейс `Payment` с методами:
    - `GetCreatedAt() int64`
    - `GetAmount() int`

4. Создайте функцию `ParsePayments`:
   - Принимает на вход путь к файлу с данными о платежах
   - Возвращает срез платежей (интерфейс `Payment`)

5. Создайте функцию `CalculateTotalSum`:
    - Принимает на вход срез платежей (интерфейс `Payment`)
    - Возвращает общую сумму платежей

6. Создайте функцию `CalculateAverageSum`:
    - Принимает на вход срез платежей (интерфейс `Payment`)
    - Возвращает среднюю сумму платежа

7. Создайте функцию `CalculatePaymentTotalIncome`:
   - Принимает на вход 
     - срез платежей (интерфейс `Payment`)
     - опциональные параметры:
       - `from` - начальная временная метка (`time.Time`)
       - `to` - конечная временная метка (`time.Time`)
   - Возвращает общую выручку (за период, если указаны параметры `from` и/или `to`)
